## “NAME” Authentication Technique
Ontological Checks (CONJUNCTION)

- [ ]  Does the approach involve one or more Authenticators?
  - 
- [ ]  Does the approach rely on the Subject’s use of these Authenticators to produce Authenticator Output?
  - 
- [ ]  Does the approach allow the Verifier to validate the Subject’s identity using this Authenticator Output?
  - 
- [ ]  Is the approach conceptual, i.e. does it independent of technical details, such as authentication data and the protocols used to transmit it?

  Hint: This should not be understood as a strict true/false assessment. There can be borderline assessmenst (e.g. an approach contains technical details to some minor extend). In such cases, an approach should still be added to the AuthN Technique catalog.
   
  - 

## Catalog Checks (DISJUNCTION)

- [ ]  Is the way how the Subject uses these Authenticators to produce Authenticator Output distinct from other AuthN Techniques in the catalog?
  - 
- [ ]  Does the AuthN Technique involve a unique Authenticator compared to other AuthN Techniques catalog?

  Hint: If the interaction style is not unique but the Authenticator is different, it should be considered a unique AuthN Technique
   
  - 
    - [ ]  If not, does the Authenticator has unique ontological constraints that allow to model the Authenticator as a unique specialization of another Authenticator?
      - 
- [ ]  Is the way how the Authenticator Output is conceptually used to verify the Subject’s identity distinct from other AuthN Techniques in the catalog?
